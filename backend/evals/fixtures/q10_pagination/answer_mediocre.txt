OFFSET and LIMIT work fine at first but get slow on deep pages, because OFFSET 500000 makes the database walk through half a million rows just to throw them away. The better approach is cursor pagination: instead of a page number, the client passes back the last ID it saw, and the query does WHERE id > last_id ORDER BY id LIMIT 50. With an index on the ID that's fast no matter how deep you go. Most big APIs like GitHub's work this way.
